```
                     │
                     ▼
```

**القنوات المدعومة:** واتساب فقط

## المشغل

### العقدة 1: Sidawi_Health_WH
```
النوع:    n8n-nodes-base.webhook
المسار:   POST /webhook/sidawi-health-v22
الغاية:   استقبال رسائل WhatsApp الواردة من بوابة WA Gateway
```

**البيانات الواردة:**
```json
{
  "body": {
    "from": "966501234567",
    "content": "كود التفعيل أو سؤال الموظف"
  }
}
```

**الإعداد:**
- لا تحتاج credentials في n8n
- سجِّل الـ Webhook URL في بوابة واتساب الخاصة بك
- الـ URL النهائي: `https://your-n8n-instance.com/webhook/sidawi-health-v22`

## التحقق

### العقدة 2: Verify_Identity
```
النوع:    n8n-nodes-base.httpRequest
الطريقة:  POST
الرابط:   /api/v1/bot/check-auth
الغاية:   التحقق من هوية المستخدم ومعرفة الإجراء المطلوب
```

**ما يُرسله:**
```json
{
  "phoneNumber": "966501234567",
  "activationCode": "ABCD1234"
}
```

**الـ Header المطلوب:**
```
x-api-key: <مفتاح machine من إعدادات التطبيق>
```

**ما يرجعه السيرفر:**
```json
{
  "authorized": true,
  "action": "activated",
  "full_name": "إبراهيم محمد",
  "deactivation_code": "STOP-XYZ9"
}
```

**قيم action الممكنة:**

| القيمة | المعنى | المسار |
|--------|--------|--------|
| `activated` | تفعيل ناجح | WA_Welcome → WA_Admin_Activation |
| `deactivated` | إيقاف يدوي | WA_Goodbye |
| `auto_deactivated` | انتهاء الجلسة تلقائياً | WA_AutoTimeout |
| `message` | المستخدم نشط يطرح سؤالاً | AI_Agent |
| `unauthorized` | كود خاطئ أو مستخدم غير مسجَّل | لا يوجد رد — الطلب يُهمَل |

> **ملاحظة V22:** على عكس V23، لا توجد عقدة `Admin_Unauthorized_Alert` في V22. محاولات الوصول غير المصرح به تُهمَل بصمت (لا رد للمستخدم، لا إشعار للمدير). إذا كنت بحاجة لهذا الإشعار، استخدم V23.

## التوجيه

### العقدة 3: Route_by_Action
```
النوع:    n8n-nodes-base.switch
الغاية:   توجيه التنفيذ للمسار الصحيح بناءً على قيمة action
```

| المخرج | القيمة | الوجهة |
|--------|--------|--------|
| 0 — تفعيل | `activated` | `WA_Welcome` |
| 1 — إيقاف يدوي | `deactivated` | `WA_Goodbye` |
| 2 — إيقاف تلقائي | `auto_deactivated` | `WA_AutoTimeout` |
| 3 — رسالة للذكاء | `message` | `AI_Agent` |

```
النوع:    n8n-nodes-base.httpRequest
الطريقة:  POST
```

### العقدة 5: WA_Goodbye
```
الغاية:   إبلاغ المستخدم بأن البوت تم إيقافه يدوياً
```

**الرسالة:**
```

لن يتم الرد على رسائلك حتى تقوم بإرسال كود التفعيل مجدداً.
```

## الذكاء الاصطناعي

### العقدة 7: AI_Agent
```
النوع:    @n8n/n8n-nodes-langchain.agent
النموذج:  Google Gemini (يُحدَّد من الـ Credential)
الغاية:   المعالجة الذكية للاستفسارات واستدعاء الأدوات المناسبة
```

**System Prompt (الموجِّه — صارم ومُقيَّد):**
```
أنت النظام الخبير للإدارة التقنية والذاتية بمديرية الصحة.
الموظف المصرَّح له: {{ full_name }}

قواعد صارمة — لا استثناءات مطلقاً:

[1] استدعاء الأداة إلزامي:
- لكل استفسار عن موظف أو إحصاءات أو مستندات، يجب استدعاء
  أداة fetch_employee_database أولاً، قبل أي رد.
- إذا لم تستدعِ الأداة، فأنت لا تملك أي بيانات ولا يحق لك الرد.
- يُحظر تماماً استخدام أي معلومة من معرفتك التدريبية.

[2] الموظف غير الموجود — قاعدة حرفية:
- الرد الوحيد المسموح به:
  "لا يوجد موظف بهذا الاسم أو الرقم في قاعدة بيانات المديرية."
- لا تضف أي كلام. لا تخمن. لا تقترح أسماء بديلة.

[3] المستندات — قاعدة صارمة:
- تُعرض فقط من حقل direct_url في بيانات الموظف.
- إذا كان فارغاً: "لا توجد مستندات مرفقة بسجل هذا الموظف."
- يُحظر اختراع روابط أو وصف مستندات غير موجودة.

[4] ملف Word:
- استخدم generate_word_link عند الطلب الصريح فقط.
- احصل على nationalId من fetch_employee_database أولاً.

[5] ملف Excel المخصص:
- استخدم export_excel_tool مع الفلاتر المناسبة.
- أرسل downloadUrl مباشرةً.

[6] نطاق الردود:
- اللغة العربية الفصيحة، مختصرة وواضحة.
- خارج نطاق بيانات المديرية:
  "أنا متخصص في بيانات موظفي المديرية فقط."
```

## إشعارات المدير

### العقدة 14: WA_Admin_Activation ← جديدة
```
النوع:    n8n-nodes-base.httpRequest
الطريقة:  POST
الرابط:   http://172.17.0.1:8082/send-text
الغاية:   إبلاغ المدير فوراً عند تفعيل مستخدم جديد للبوت
مُشغَّلة من: WA_Welcome (بعد إرسال رسالة الترحيب)
```

**ما يُرسله:**
```json
{
  "number": "رقم_المدير",
  "message": "✅ تفعيل جديد للبوت\n\n👤 الاسم: {{ full_name }}\n📞 الرقم: {{ from }}\n⏰ الوقت: {{ timestamp }}"
}
```

**⚠️ إعداد إلزامي — خطوة واحدة:**
1. افتح الورك فلو في n8n
2. اضغط على عقدة `WA_Admin_Activation`
3. في حقل `number` بدّل `9671XXXXXXXXXX` برقم هاتف المدير الفعلي (أرقام فقط)
4. احفظ الورك فلو

**الرسالة التي يستلمها المدير:**
```
✅ تفعيل جديد للبوت

👤 الاسم: إبراهيم محمد
📞 الرقم: 9671XXXXXXXXX
⏰ الوقت: 19/03/2026، 14:30

💡 المستخدم أصبح نشطاً في النظام.
```

> **للإشعارات المتقدمة (أخطاء + غير مصرح + تنظيف + تيليغرام):** استخدم V23.

## إعداد الورك فلو

### الخطوات الكاملة:

#### 1. استيراد الورك فلو
1. حمِّل ملف `Sidawi_AI_Health_V22.json` من التطبيق:
   **الإعدادات → تبويب الإشعارات → تحميل V22**
2. في n8n: **Workflows → Import from File**
3. اختر الملف واستورده

#### 2. إعداد Google Gemini API
1. اذهب إلى **Credentials → New → Google Gemini API**
2. أدخل مفتاح API من Google AI Studio
3. ارجع إلى عقدة `Gemini_Model` واختر الـ Credential الذي أنشأته

#### 3. تحديث رابط التطبيق ومفتاح API
- في عقدة `Verify_Identity`: غيّر الرابط لرابط تطبيقك الفعلي
- في جميع عقد HTTP Request للتطبيق: حدِّث قيمة `x-api-key` بمفتاح machine من **الإعدادات → مفاتيح API**

#### 4. تحديث رقم المدير
- في عقدة `WA_Admin_Activation`: غيّر `9671XXXXXXXXXX` برقم المدير الفعلي

#### 5. إعداد بوابة واتساب
- تأكد أن بوابة WA تعمل على `http://172.17.0.1:8082`
- سجِّل Webhook URL الخاص بـ n8n في بوابتك
- تنسيق إرسال الرسائل: `POST /send-text` بـ `{ "number": "...", "message": "..." }`

#### 6. تفعيل الورك فلو
- اضغط على **Activate** في أعلى يمين الورك فلو
- اختبر بإرسال كود التفعيل عبر واتساب

## الفروقات

### V22 مقابل V23

| الميزة | V22 | V23 |
|--------|-----|-----|
| القنوات | واتساب فقط | واتساب + تيليغرام |
| تطبيع البيانات | لا يوجد (مباشر) | Normalize_WA + Normalize_TG |
| معالجة الأخطاء | لا يوجد | Check_Service_Error + Source_Error_Router |
| إشعار الأخطاء للمدير | لا يوجد | Admin_Error_Alert |
| إشعار الوصول غير المصرح | لا يوجد | Admin_Unauthorized_Alert |
| إشعار التفعيل للمدير | WA_Admin_Activation (مباشر) | Admin_Activation_Alert (عبر API) |
| تقرير التنظيف الساعي | لا يوجد | Admin_Cleanup_Report |
| تسجيل المحادثات | لا يوجد | Log_Conversation |
| أداة الإحصاءات السريعة | لا يوجد | get_employee_stats |
| إجمالي العقد | 14 | 37 |
| مستوى التعقيد | بسيط | متقدم |

### متى تستخدم V22؟
- إذا كانت القناة الوحيدة هي واتساب
- إذا كنت تريد إعداداً سريعاً وبسيطاً
- إذا كان تسجيل المحادثات ليس ضرورياً

### متى تستخدم V23؟
- إذا أردت دعم تيليغرام أيضاً
- إذا أردت إشعارات كاملة للمدير (أخطاء + غير مصرح + تقارير)
- إذا أردت تسجيل جميع المحادثات في قاعدة البيانات
- إذا كنت تدير نظاماً في بيئة إنتاجية احترافية
